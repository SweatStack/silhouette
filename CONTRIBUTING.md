# Contributing to Silhouette

Thanks for your interest in contributing to Silhouette! We welcome contributions of all kinds: bug reports, feature ideas, documentation improvements, and code.

## Getting started

1. Fork the repository and clone your fork.
2. Create a new branch for your changes.
3. Make your changes and ensure tests pass.
4. Open a pull request with a clear description of what you've done and why.

## Bug reports and feature requests

Good bug reports are incredibly helpful. Please include:

- A clear description of the problem or idea.
- Steps to reproduce the issue, if applicable.
- What you expected to happen and what actually happened.

## Pull requests

- Keep pull requests focused and reasonably scoped. Smaller, well-explained PRs are much easier to review and merge.
- Include context in your PR description: what problem does it solve, and how did you approach it?
- Make sure your code is tested and follows the existing style of the project.

## A note on quality

We value contributions that show thought and effort. When opening issues or pull requests, please take the time to understand the codebase, write clear descriptions, and make sure your changes are something you've personally worked through. Bulk-generated or low-effort submissions (such as AI-generated issues or large AI-generated PRs submitted without meaningful review) are not a good fit for this project and will be declined.

This isn't about the tools you use, it's about showing up with care and intention.

## Code of conduct

Be kind, be respectful, and assume good intent. We're all here to build something useful together.

## Questions?

If you're unsure about anything, feel free to open an issue and ask. We're happy to help you get started.
